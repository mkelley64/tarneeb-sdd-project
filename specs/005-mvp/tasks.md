# Tarneeb iOS MVP 005 Implementation Tasks

This task list is derived from `requirements.md`, `design.md`, and
`design-tokens.md` in `specs/005-mvp`. It is an implementation plan only; no
code is included here.

Acceptance criteria are referenced as `PRD-### AC#`, where `AC#` is the
acceptance criterion order within that product requirement. Non-functional
requirements are referenced as `NFR-###`.

All concrete color, typography, title-shadow, dealer-badge, bidding-area,
post-bidding summary, current-highest bid value, bid-selector, South Tarneeb
suit selector, South `Bid` button, bid fade/color animation, bidding-area fade
animation, South interim fanned-back stack, South reveal flip animation,
undealt-deck stack, and layout values
must come from
`specs/005-mvp/design-tokens.md`. If implementation needs a value not
represented by that token source, update the token spec before using the value
in app or test code.
